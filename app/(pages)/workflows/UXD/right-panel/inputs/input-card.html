Input Card Component

Detailed mockup of the single input card component showing all fields, type variations, and interaction states.

Card 1: String Type - Complete AC-F.1 through F.6, AC-9.3, AC-T.1
*

Helpful description for this input parameter

This value will be used if not provided when the workflow is invoked. For string type, enter any text value.

String Type Features:

  • Name field: Text input with monospace font (AC-F.1)
  • Type selector: Dropdown with 5 options (AC-F.2)
  • Required checkbox: Toggles required status (AC-F.3)
  • Description: Optional text input (AC-F.4)
  • Default value: Text input for string type (AC-T.1)
  • Delete button: Visible on hover, red on hover (AC-F.6)
Card 2: Number Type AC-T.2

Numeric value. Accepts integers and decimals based on step.

Number Type Features (AC-T.2):

  • Default value field changes to type="number" input
  • Browser native number input with increment/decrement controls
  • Optional min/max validation (implementation detail)
  • Supports both integers and floating-point numbers
Card 3: Boolean Type AC-T.3

When boolean type is selected, default value becomes a checkbox. Checked = true, Unchecked = false.

Boolean Type Features (AC-T.3):

  • Default value field transforms into a checkbox UI
  • Checkbox checked = true, unchecked = false
  • Visual indication shows current state clearly
  • Larger checkbox (20px) for better UX
Card 4: Optional Input (Not Required)
(optional)

Optional Input Visual Differences:

  • Required checkbox is unchecked
  • Label shows muted gray color instead of dark
  • No red asterisk indicator
  • Input can be omitted when workflow is invoked
Card 5: Focused State (Editing)

Focus State Visual Indicators:

  • Card border changes to blue-500 (2px)
  • Ring effect around card (blue-500/20 opacity)
  • Active input has blue background (bg-blue-50)
  • Blue border on focused input field
  • Shadow increases for depth perception
Card 6: Default Value Field Focused AC-9.4

Saved

Value auto-saves when you click away (blur event)

Default Value Focus Behavior (AC-9.4):

  • Field receives focus with blue border (border-blue-500)
  • Background becomes more prominent (bg-blue-100)
  • On blur: Value persists immediately to store
  • Visual feedback: Optional "Saved" indicator (implementation detail)
  • No explicit save button needed - auto-saves
Card 7: Array/Object Type (Future - AC-T.4)

Array/Object default values coming in future phase. Will support JSON input with validation.

Future Enhancement (AC-T.4):

  • Array/Object types will support JSON textarea for default values
  • JSON syntax validation
  • Syntax highlighting (optional enhancement)
  • Currently disabled/placeholder state

Input Card Component Summary

Core Fields:

  • • Name input (AC-F.1) - Text, monospace, required
  • • Type selector (AC-F.2) - Dropdown, 5 options
  • • Required checkbox (AC-F.3) - Toggles required status
  • • Description field (AC-F.4) - Optional text input
  • • Default value (AC-F.5, AC-9.3) - Type-aware input
  • • Delete button (AC-F.6) - Hover to reveal

Type-Specific Defaults:

  • • String: Text input (AC-T.1)
  • • Number: Number input (AC-T.2)
  • • Boolean: Checkbox (AC-T.3)
  • • Array/Object: JSON textarea (AC-T.4 - future)

Interaction States:

  • • Default: Standard gray borders, white background
  • • Hover: Shadow increase, delete button opacity 1
  • • Focus: Blue border, ring effect, blue background on active field
  • • Editing: Immediate updates, auto-save on blur (AC-9.4)
  • • Error: Red border (future - validation)
